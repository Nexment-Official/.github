# Contributing to Nexment

First of all, thank you for your interest in contributing to Nexment! ❤️

Whether you're reporting a bug, suggesting a feature, improving documentation, testing releases, creating tutorials, or helping other community members, your support helps improve the Nexment ecosystem.

---

# Current Contribution Policy

At this stage, **Nexment is not accepting code contributions**.

The ecosystem is still in its early stages, and the core architecture is evolving rapidly. To maintain a consistent design, development is currently handled by the Nexment maintainers.

As Nexment grows and larger projects mature, code contributions will gradually become part of the development process. When that time comes, contribution guidelines and coding standards will be published for each project.

Until then, there are many valuable ways you can contribute.

---

# Ways You Can Contribute

We welcome contributions in areas such as:

- 🐞 Reporting bugs
- 💡 Suggesting new features or improvements
- 📖 Improving documentation
- 🌍 Translating documentation
- 🧪 Testing releases and reporting issues
- 🎨 Creating icons, artwork, graphics, or other design assets
- 🎥 Creating tutorials, articles, videos, or other educational content
- 💬 Participating in discussions and helping community members

Every contribution helps make Nexment better.

---

# Before Opening an Issue

Before creating an issue, please:

- Search existing issues to avoid duplicates.
- Read the available documentation.
- Make sure you've reproduced the issue.
- Use the appropriate issue template whenever possible.

Providing detailed information helps us resolve issues more efficiently.

---

# Feature Requests

We welcome ideas that improve Nexment.

Before submitting a feature request:

- Check whether a similar suggestion already exists.
- Clearly explain the problem you're trying to solve.
- Describe your proposed solution and why it would be valuable.

Please understand that not every feature request will be accepted.

---

# Design Decisions

Nexment follows a long-term vision focused on performance, simplicity, privacy, and consistency.

Community feedback plays an important role in shaping the ecosystem. However, architectural and product decisions are made by the maintainers to ensure every project remains consistent with Nexment's long-term goals.

---

# Communication

Please keep discussions respectful, constructive, and professional.

When interacting with maintainers or other community members:

- Be respectful and patient.
- Stay focused on technical discussions.
- Accept feedback gracefully.
- Respect maintainer decisions.

Healthy discussions are encouraged, but personal attacks, hostility, or disrespectful behavior are not.

---

# Future Code Contributions

Code contributions will open as larger Nexment projects mature and the development process becomes ready for community collaboration.

When that happens:

- Every accepted contribution will receive full credit.
- Project-specific contribution guides will be published.
- Coding standards and development workflows will be documented.

---

# Recognition

We deeply appreciate every contribution to Nexment.

Whether you've reported bugs, improved documentation, tested releases, translated content, created educational resources, or contributed code in the future, your work helps move the ecosystem forward.
